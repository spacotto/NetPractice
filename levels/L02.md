## How to Solve Level 2

### Goal 1: Host "Computer B" needs to communicate with host "Computer A"

To achieve Goal 1, we must connect `Host A` (Computer A) to `Host B` (Computer B). **Computer A (`A1`)** IP is editable, but its **subnet mask** is locked at `255.255.255.224`. This represents a `/27` network mask. On the other hand, **Computer B (`B1`):** IP is locked at `192.168.141.222`, but its **subnet mask** is editable and currently contains the illegal placeholder `255.255.255.32`. Because Computer A's mask is unchangeable and Computer B's IP is frozen at `.222`, we must **align both devices inside the specific `/27` subnet block** that contains the address `.222`. 

A `/27` mask slices network ranges into sequential blocks of **32 addresses**. The block covering `.222` runs from `.192` to `.223`:
* **Network ID:** `192.168.141.192` (Unusable)
* **Usable Host Range:** `192.168.141.193` to `192.168.141.222`
* **Broadcast Address:** `192.168.141.223` (Unusable)

To fix the network connection:
* [x] **STEP 1:** Correct `Computer B`'s broken subnet mask to match Computer A's valid network mask: **`255.255.255.224`**.
* [x] **STEP 2:** Edit `Computer A`'s IP address to place it into this exact same `.192` block. Change it from `192.168.141.1` to any available slot in the block (such as **`192.168.141.200`**).

**Example Solution Configuration:**
| Device | Interface | IP Address | Subnet Mask |
| :--- | :--- | :--- | :--- |
| **Computer A** | `A1` | `192.168.141.200` *(Modified)* | `255.255.255.224` *(Locked)* |
| **Computer B** | `B1` | `192.168.141.222` *(Locked)* | `255.255.255.224` *(Modified)* |

### Goal 2: Host "Computer D" needs to communicate with host "Computer C"

To achieve Goal 2, we must connect `Host C` (Computer C) to `Host D` (Computer D). **Computer C (`C1`)** mask is locked at `255.255.255.252` (`/30`). Its IP defaults to `127.0.0.1`. On the other hand, **Computer D (`D1`)** mask is locked at `/30` (which is functionally identical to `255.255.255.252`). Its default IP is `127.0.0.4`. 

A `/30` mask reserves 30 bits for the network identifier, leaving only 2 bits for hosts (`32 - 30 = 2`). This configuration yields a tiny block size of 4 total addresses (`2^2 = 4`). 

In real-world networking, the entire IP address range from `127.0.0.0` to `127.255.255.255` is strictly reserved for **loopback addresses** (commonly known as **localhost**). When a computer sends a packet to a `127.x.x.x` address, the operating system's network stack intercepts it immediately and loops it right back to itself inside the machine's memory. A loopback packet never leaves the **network interface card (NIC)**. It is physically impossible to route a packet starting with `127` across an external cable or switch link to another machine.

In other words, we have to change from `127.0.0.1` to `192.168.141.1` because `127.0.0.1` **cannot travel across network links to another computer**. Moreover, the NetPractice simulator dynamically rewrites the level template to force both goals onto a uniquely randomised subnet: `192.168.141.0`.

For the `192.168.141.0` micro-segment, the boundary lines are mathematically defined as:
* **Network ID:** `192.168.141.0` (Unusable)
* **First Usable Host IP:** `192.168.141.1`
* **Second Usable Host IP:** `192.168.141.2`
* **Broadcast Address:** `192.168.141.3` (Unusable)

To clear the level objectives, we must manually point both hosts to the only two valid host slots existing inside this `/30` boundary:
* [x] **STEP 1:** Set `Computer C`'s IP address field to **`192.168.141.1`**.
* [x] **STEP 2:** Set `Computer D`'s IP address field to **`192.168.141.2`**.

**Solution Configuration Table:**
| Device | Interface | IP Address | Subnet Mask |
| :--- | :--- | :--- | :--- |
| **Computer C** | `C1` | `192.168.141.1` *(Modified)* | `255.255.255.252` *(Locked)* |
| **Computer D** | `D1` | `192.168.141.2` *(Modified)* | `/30` *(Locked)* |

## Theoretical Concepts

### The Strict Limitations of a /30 Subnet

A `/30` (or `255.255.255.252`) network mask is the smallest possible functional subnet block for host-to-host links. Out of its 4 total addresses, 2 are automatically consumed by the network infrastructure rules (Network ID and Broadcast Address), leaving exactly 2 addresses available for endpoints.

### Multi-Octet Randomisation Mechanics

During a defence evaluation, NetPractice generates a dynamic token variable (like the `141` seen here) to ensure that students understand the mathematical formulas behind subnets rather than memorising static values. No matter what number gets swapped into that third octet position, the host separation logic remains identical.
