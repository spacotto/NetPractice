## How to Solve Level 2

### Goal 1: Host "Computer B" needs to communicate with host "Computer A"

* **Computer A (`A1`):** IP is editable, but its Subnet Mask is locked at `255.255.255.224`. This represents a `/27` network mask.
* **Computer B (`B1`):** IP is locked at `192.168.141.222`, but its Subnet Mask is editable and currently contains the illegal placeholder `255.255.255.32`.

Because Computer A's mask is unchangeable and Computer B's IP is frozen at `.222`, we must align both devices inside the specific `/27` subnet block that contains the address `.222`. 

A `/27` mask slices network ranges into sequential blocks of 32 addresses. The block covering `.222` runs from `.192` to `.223`:
* **Network ID:** `192.168.141.192` (Unusable)
* **Usable Host Range:** `192.168.141.193` to `192.168.141.222`
* **Broadcast Address:** `192.168.141.223` (Unusable)

To fix the network connection:
* [x] **STEP 1:** Correct `Computer B`'s broken subnet mask to match Computer A's valid network mask: **`255.255.255.224`**.
* [x] **STEP 2:** Edit `Computer A`'s IP address to place it into this exact same `.192` block. Change it from `192.168.141.1` to any available slot in the block (such as **`192.168.141.200`**).

**Corrected Solution Configuration:**
| Device | Interface | IP Address | Subnet Mask |
| :--- | :--- | :--- | :--- |
| **Computer A** | `A1` | `192.168.141.200` *(Modified)* | `255.255.255.224` *(Locked)* |
| **Computer B** | `B1` | `192.168.141.222` *(Locked)* | `255.255.255.224` *(Modified)* |

### Goal 2: Host "Computer D" needs to communicate with host "Computer C"

* **Computer C (`C1`):** Mask is completely locked at `255.255.255.252` (`/30`). Its IP defaults to `127.0.0.1`.
* **Computer D (`D1`):** Mask is locked at `/30` (which is functionally identical to `255.255.255.252`). Its default IP is `127.0.0.4`. 
* Both IP fields are open for editing (`ip_edit: true`).

A `/30` mask leaves only 2 bits for hosts, breaking the network into tiny 4-IP blocks. 
* **Subnet Block 0:** Covers `127.0.0.0` through `127.0.0.3`. Usable host positions are `.1` and `.2`. 
* **Subnet Block 1:** Starts exactly at `127.0.0.4` (which is the Network ID for that next segment).

Because Computer C is sitting on `127.0.0.1` (Block 0) and Computer D is sitting directly on a Network ID identifier (`127.0.0.4`), they cannot communicate. We must manually pull Computer D down into Computer C's active network block.

* [x] **STEP 1:** Keep `Computer C` on its default configuration (`127.0.0.1`).
* [x] **STEP 2:** Edit `Computer D`'s IP address field, changing it from `127.0.0.4` to the only other available host slot in Block 0: **`127.0.0.2`**.

**Corrected Solution Configuration:**
| Device | Interface | IP Address | Subnet Mask |
| :--- | :--- | :--- | :--- |
| **Computer C** | `C1` | `127.0.0.1` *(Default)* | `255.255.255.252` *(Locked)* |
| **Computer D** | `D1` | `127.0.0.2` *(Modified)* | `/30` *(Locked)* |

## Theoretical Concepts

### Subnet Counting (The /27 Increment Block)

A `/27` network uses 27 bits for the network identifier, leaving 5 bits for host assignment ($32 - 27 = 5$). 
* $2^5 = 32$, which means subnets strictly increment by jumps of 32 values.
* To discover a host's boundary, find the closest multiple of 32 below that IP address. For `222`, the closest lower multiple is $32 \times 6 = 192$. This mathematically confirms that `.192` is the Network boundary start point.
