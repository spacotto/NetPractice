## How to Solve Level 10

### Analyse the Network Constraints (Top-Down Planning)

Identify the fixed constraints:

* Examine the `Internet` interface and its routing table: the destination route will define the specific public IP block available for your external-facing router interface.
* Identify any locked Subnet Masks (Level 10 is the final challenge and requires building a complete, efficient network with correct IP allocation, routing, and no wasted space. ### Determine the Router-to-Router Transit Links

In complex topologies, routers must communicate with each other before reaching the hosts.

* Allocate the smallest possible subnet for router-to-router connections to conserve IP space. A `/30` mask (`255.255.255.252`) is ideal because it provides exactly two usable IPs—one for each router interface.
* Ensure the default routes (`0.0.0.0/0`) on internal routers point to the next hop closest to the Internet.

### Allocate Host Networks (VLSM)

Once transit links are established, distribute the remaining IP blocks to the host networks using Variable Length Subnet Masking (VLSM).

* Calculate the required network size for each switch cluster. If a host interface has a locked `255.255.255.240` (`/28`) mask, you must reserve a block of 16 addresses.
* Assign the first usable IP to the router's local interface (acting as the default gateway) and the subsequent IP(s) to the connected hosts.
* Ensure there is no overlap between subnets.

### Configure Routing Tables

Establish the remaining routes to ensure traffic flows seamlessly end-to-end between all devices.

* **Hosts:** Point all host default routes (`0.0.0.0/0`) to the IP address of their immediate router's local interface.
* **Routers:** Configure static routes or summary routes to point traffic destined for specific subnets to the correct router interface.
* Verify the routing paths using the logs provided at the bottom of the NetPractice interface.

## Theoretical Concepts

### IP Addressing and Subnetting

An IP address identifies devices on a network, while the subnet mask determines which portion of the address represents the network and which represents the host. In this level, correctly dividing the address space into appropriately sized blocks (subnetting) is critical to avoid IP overlap and ensure all devices can communicate.

### The Default Gateway

When a host needs to communicate outside its local subnet (e.g., to the Internet or another local network), it forwards the data to its default gateway. In TCP/IP addressing, the destination `0.0.0.0/0` in the routing table represents this default path, capturing any traffic that does not have a more specific route.
