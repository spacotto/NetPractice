## How to Solve Level 10

### Configure Switch One Subnet

Change `interface H21` and `interface H11` masks to match `interface R11` (`255.255.255.128`, `/25`). Then, change `interface H21` IP to match `interface R11` and `interface H11` (e.g., `159.170.184.3`).

### Configure Router Two Subnet

Change `interface R23` IP and mask to match `interface H41` mask (`255.255.255.192`, `/26`) and `host H4` gateway.

Then, since local host IPs are not allowed and you need to establish an internet connection, change the `interface R22` and `interface H31` IPs to match the other side of the subnet. Finally adjust the routing table `host H3` to match the gateway with `interface R22` IP.

### Configure Router One

Change `router R1` route (destination) to default (`0.0.0.0/0`). Then, align the mask of `interface R13` to `interface R21` masks (`255.255.255.252`).

### Determine Internet Route (Destination)

Moving `host 3` from the private `10.0.0.0` network into the available `[a].[b].[c].x` public block is necessary because private IPs cannot be routed across the internet. By unifying all devices under the same public address space, the Internet router no longer drops packets destined for a local IP. This consolidation reliably allows all return traffic to flow back into your network without routing conflicts.

Then, expanding the Internet’s return route mask from a restrictive `/31` to a broader `/24` is required because a `/31` only covers two specific IP addresses, leaving the rest of your devices invisible. A `/24` mask widens the boundary to encompass the entire `[a].[b].[c]` block used by your hosts. This ensures that any external reply destined for your network is successfully identified and forwarded to your edge router.

## Theoretical Concepts

### Default Route Aggregation on the Edge

In complex topologies where internal networks utilise entirely different address blocks (such as a private `10.0.0.0/8` network working alongside an `[a].[b].[c].0/24` block), an edge routing environment must provide an elegant way for external entities to return traffic. By changing the Internet's edge target route to a default path (`0.0.0.0/0`) pointing back to our public router interface, the Internet successfully offloads routing decisions to our edge gateway, ensuring all internal subnets remain reachable.

### Subnet Mask Peering across Point-to-Point Links

For two layer-3 devices to successfully forward packets across a shared physical or virtual link, their connecting interfaces must agree entirely on the network boundary. If one router interface is locked to a `/30` mask (`255.255.255.252`), keeping a broad mask like `255.255.255.0` on the opposite side will result in mismatched broadcast domains. Ensuring both sides use an identical subnet mask prevents packet drops caused by routing engine calculation mismatches.
