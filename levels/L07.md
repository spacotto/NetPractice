## How to Solve Level 7

### Analyse the Locked Router Interfaces (VLSM Planning)

In this topology, you are given a single `/24` network space (`[a].198.14.x`) that must be divided across three distinct network segments using Variable Length Subnet Masking (VLSM). 
* Examine `Router R1`. It has two locked IP addresses: `R11` (`.1`) and `R12` (`.254`). If they both share a standard `/24` mask, their networks overlap and the router will fail. 
* Isolate the router-to-router link (`R12` to `R21`). Because `R12` is permanently locked to `.254`, the most efficient way to isolate it is by using a `/30` point-to-point mask (`255.255.255.252`). The `/30` block containing `.254` strictly spans from `.252` to `.255`.

### Configure the Interfaces

Divide the remaining IP space into valid blocks for the host networks, ensuring no subnets overlap.
* **Network R1-R2 (The Bridge):** Apply the `/30` mask (`255.255.255.252`) to `R12`. Since the usable IPs in this block are `.253` and `.254`, change `R21`'s IP to `.253` and apply the same `/30` mask.
* **Network A (Host A to R1):** `R11` is locked at `.1`. Apply a `/25` mask (`255.255.255.128`) to create a subnet spanning `.0` to `.127`. Set `A1` to match the mask, safely leaving its IP at `.2`.
* **Network C (Host C to R2):** Create a third block that avoids `.0-.127` and `.252-.255`. Use a `/26` mask (`255.255.255.192`) to generate a clean block from `.128` to `.191`. Change `R22`'s IP to `.129` and `C1`'s IP to `.130`. (Note: `C1` starts with `.1`, which is invalid as it belongs to Network A). Apply the `/26` mask to both.

### Configure the Routing Tables

In a linear topology, every device simply needs to point its "default route" to the very next hop in the chain. 
* Change the destination for all routing tables to `0.0.0.0/0`.
* **Host A:** Point the Gateway to `R11`'s exact IP (`.1`).
* **Host C:** Point the Gateway to `R22`'s exact IP (`.129`).
* **Router R1:** To reach C's network, point the Gateway to `R21`'s IP (`.253`).
* **Router R2:** To reach A's network, point the Gateway to `R12`'s IP (`.254`).

## Theoretical Concepts

### Point-to-Point Links (/30)

When connecting two routers directly together, deploying a large subnet like a `/24` wastes 252 usable IP addresses. A `/30` subnet mask (`255.255.255.252`) provides exactly 4 total addresses: a Network ID, a Broadcast Address, and precisely 2 usable IPs for the two router interfaces. This is the industry standard for securing and optimizing point-to-point infrastructure links.

### Linear Routing Flow

When dealing with a simple chain of networks, complex routing tables are unnecessary. By utilizing the default route (`0.0.0.0/0`) on every device, you create a seamless "bucket brigade" where each device simply hands unrecognized traffic to the next device in line, relying on the routers to figure out the ultimate destination.
