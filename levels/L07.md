## How to Solve Level 7

>[!WARNING]
>Due to the flexibility of this level, there are many possible solutions. I am about to explain the simplest from my point of view.

### Configure the Interfaces

In this topology, you are given a single `/24` network space (`[a].198.14.x`) that must be divided across three distinct network segments using Variable Length Subnet Masking (VLSM). However, slicing the network with `/30` masks makes the math much easier. Thus, set all masks to `/30` (or `255.255.255.252`).

A `/30` mask has only **four** possible IPs, only **two** of which are usable. Given the level topology, here are the possible solutions for each one of the three slices.

**Interface A1 - Interface R11**:
* Network Address: `[a].198.14.0`
* Broadcast Address: `[a].198.14.3`
* Usable Host IP Range:	`[a].198.14.1` and `[a].198.14.2`

**Interface R12 - Interface R21**:
* Network Address: `[a].198.14.252`
* Broadcast Address: `[a].198.14.255`
* Usable Host IP Range:	`[a].198.14.253` and `[a].198.14.254`

>[!TIP]
>Since we used the first and last possible slices, for **Interface R22 - Interface C1** subnet, anything in the middle is acceptable. Here are the 2 easiest solutions to calculate.

Second slice (count the next from the first)
* Network Address: `[a].198.14.4`
* Broadcast Address: `[a].198.14.7`
* Usable Host IP Range:	`[a].198.14.5` and `[a].198.14.6`

Second last slice (count in reverse from the last)
**Interface R22 - Interface C1**:
* Network Address: `[a].198.14.248`
* Broadcast Address: `[a].198.14.251`
* Usable Host IP Range:	`[a].198.14.249` and `[a].198.14.250`

### Configure the Routing Tables

In a linear topology, every device simply needs to point its "default route" to the very next hop in the chain. 
* `dev.non-real.net` points to `tech.non-real.net` (`interface R11` IP)
* `tech.non-real.net` points to `adm.non-real.net` (`interface R21` IP)
* `adm.non-real.net` points to `tech.non-real.net` (`interface R12` IP)
* `accounting.non-real.net` points to `adm.non-real.net` (`interface R22` IP)

## Theoretical Concepts

### Point-to-Point Links (/30)

When connecting two routers directly together, deploying a large subnet like a `/24` wastes 252 usable IP addresses. A `/30` subnet mask (`255.255.255.252`) provides exactly 4 total addresses: a Network ID, a Broadcast Address, and precisely 2 usable IPs for the two router interfaces. This is the industry standard for securing and optimizing point-to-point infrastructure links.

### Linear Routing Flow

When dealing with a simple chain of networks, complex routing tables are unnecessary. By utilizing the default route (`0.0.0.0/0`) on every device, you create a seamless "bucket brigade" where each device simply hands unrecognized traffic to the next device in line, relying on the routers to figure out the ultimate destination.
