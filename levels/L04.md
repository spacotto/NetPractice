## How to Solve Level 4

### Analyse the Router Constraints

A router cannot have overlapping subnets on its interfaces. Examine the locked interfaces on `My_Gate` (Router `R`) to determine the available IP space for the rest of the network:
*   Observe `Interface R2`, which has a locked IP of `[a].[b].[c].1` and a locked mask of `255.255.255.128` (`/25`). Recognise that this configuration consumes the IP range from `.0` to `.127`.
*   Observe `Interface R3`, which has a locked IP of `[a].[b].[c].244` and a locked mask of `255.255.255.192` (`/26`). Recognise that this configuration consumes the IP range from `.192` to `.255`.
*   Subtract those two blocks from the `[a].[b].[c].x` network to locate the gap of unassigned addresses from `.128` to `.191`.

### Configure the Switched Network

Ensure `Host A` (`A1`), `Host B` (`B1`), and `Router R` (`R1`) share the same subnet, as they are all connected to `Switch-1` and must communicate to satisfy the level's goals.
*   Note that `Interface A1` has a locked IP of `[a].[b].[c].132`. Confirm this address correctly falls inside the available `.128` to `.191` gap.
*   Apply a `/26` subnet mask (`255.255.255.192`) to all three interfaces on the switch to perfectly fit this gap without overlapping the other router interfaces. Alternatively, use `A1`'s default `/28` mask (`255.255.255.240`), provided it is applied identically to `B1` and `R1`. 
*   Update `Interface B1`'s first three octets to exactly match `A1`. Set the fourth octet to a valid host within the gap, such as `.133`, because its starting IP contains an invalid third octet `[d]` and a fourth octet of `.193` that falls into `R3`'s territory.
*   Change `Interface R1`'s final octet to a valid host in the gap, such as `.190`. The default IP ends in `.91`, which overlaps with `R2`'s `.0 - .127` network.

>[!TIP]
>While a router mathematically accepts any IP within the usable host range (such as `.134`), always assign the default gateway to either the very first or the very last usable address in the subnet (such as `.190`). Tucking the gateway at the absolute boundary is an industry standard that leaves a clean, **contiguous block of addresses completely uninterrupted for servers or DHCP distribution**. Adhere to this convention to prevent accidental IP conflicts and drastically simplify long-term network management.

## Simulator Limitations vs. Real-World Routing

Understand why the NetPractice simulator may accept technically incorrect subnet masks (like `/23` or `/25`), even though only `/26` is architecturally valid.

### The Simulator's Blind Spot

Note that the validation script only verifies local connectivity. To pass the goals, it simply checks if `Host A`, `Host B`, and `Interface R1` share a subnet and can ping each other across the switch. It **does not** test whether Router `R` can successfully route packets out of its other interfaces (`R2` and `R3`). Because of this, the simulator allows you to assign subnets to `R1` that illegally overlap with `R2` and `R3`.

### Mask Behaviour Analysis

Analyse how different masks behave on Interface `R1` (assuming an IP in the `.132` range):

* **/23 (`255.255.254.0`):** Creates a massive subnet of 510 hosts. This completely swallows both `R2`'s network and `R3`'s network. A physical router will throw an "Overlapping Subnet" error and refuse to apply the IP.
* **/25 (`255.255.255.128`):** Creates a subnet from `.128` to `.255`. This successfully avoids `R2` (which stops at `.127`), but it completely swallows `R3` (which spans `.192` to `.255`). This causes a routing table conflict.
* **/26 (`255.255.255.192`):** Creates a subnet exactly from `.128` to `.191`. It fits perfectly between `R2` and `R3` without overlapping either.

> [!IMPORTANT]
> While the simulator passes the `/23` and `/25` masks due to incomplete validation, the `/26` (`255.255.255.192`) mask is the only answer that obeys **Variable Length Subnet Masking (VLSM)** rules and allows a **real-world router** to maintain three distinct, functional networks.

## Theoretical Concepts

### Router Subnet Isolation

A Layer 3 router connects different networks together. Keep every active interface on the router assigned to a mathematically distinct subnet so the router knows where to send packets. Understand that overlapping IP ranges will cause routing table conflicts and lead to packet forwarding failures.

### Filling the Subnet Gaps

Partition a single large network block (like a `/24`) into smaller, differently sized blocks using Variable Length Subnet Masking (VLSM). Note how in this level, a `/25` takes the lower half of the available addresses, a `/26` takes the top quarter, and the middle quarter is left perfectly open for a final `/26` block.
