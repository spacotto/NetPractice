## How to Solve Level 6

### Analyse the Local Subnet

Establish a valid connection between `webserv.non-real.com` (Host A) and `gate.non-real.com` (Router R) across the switch.
* Examine `Interface A1`, which has a locked IP of `[a].[b].[c].227`.
* Examine `Interface R1`, which has a locked mask of `255.255.255.128` (`/25`). 
* Because `.227` falls into the upper half of a `/24` network, the valid host range for this `/25` block is `.129` to `.254`. 
* Change `Interface A1`'s subnet mask to `255.255.255.128` to match the router.
* Leave `Interface R1`'s IP address at `[a].[b].[c].254`, which correctly serves as the last usable IP in this block.

### Configure Internal Routing (Host A & Router R)

Configure the routing tables so that traffic can flow from Host A, through Router R, and out to the Internet.
* **Host A:** The destination `0.0.0.0/0` is already correct. However, the default Gateway `[a].[b].[c].1` is invalid because it belongs to the lower `/25` block. Change the Gateway to point to `Interface R1`'s exact IP: `[a].[b].[c].254`.
* **Router R:** The router needs a path to forward unknown traffic to the Internet. Change the invalid destination `10.0.0.0/8` to the default route `0.0.0.0/0`. The Gateway is locked correctly to the Internet's IP (`163.172.250.1`).

### Configure External Return Routing (Internet)

For two-way communication to work, the Internet must know how to send return packets back to Host A's specific internal network.
* Examine the routing table for `Internet`. The Gateway correctly points to Router R's public-facing interface (`163.172.250.12`). 
* The destination `[a].[b].[c].0/31` is incorrect. It must encompass Host A's entire network block. Change the destination to `[a].[b].[c].128/25`.

## Theoretical Concepts

### Internet Routing and Return Paths

When a packet leaves a local network and travels to the Internet, the routers along the way must know how to send the response back. In this simulation, you manually configure the "Internet" routing table to point your specific subnet (`[a].[b].[c].128/25`) back to your router's public IP address. In the real world, this is handled dynamically by routing protocols (like BGP) or bypassed entirely using NAT (Network Address Translation).

### Default Routes (0.0.0.0/0)

Applying `0.0.0.0/0` to a router tells it to act as a gateway of last resort. If Router R receives a packet destined for `8.8.8.8` (Google's DNS server) and does not have a specific route for the `8.0.0.0` network, it will forward the packet to the `0.0.0.0/0` gateway, which points directly to the ISP/Internet.
