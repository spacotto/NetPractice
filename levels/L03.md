## How to Solve Level 3

### Determining the Master Subnet

Because `Host C`'s subnet mask is unchangeable at `255.255.255.128` (`/25`), it forces the **entire switched network segment to adopt a `/25` architecture**: change `interface A1` and `interface B1` masks to `interface C1` mask (`255.255.255.128`).

### Align IPs

| Interface | IP |
| :--- | :--- |
| A1 | `104.[a].192.125` (*Locked*) |
| B1 | `127.[b].42.42` |
| C1 | `104.[a].192.260` |

Since all the masks are now `255.255.255.128`, align the first 3 octets to match `Interface A1`: change `Interface B1` and `Interface C1` to `104.[a].192.x`

### Determining the Correct IP (Last Octet)

| Interface | IP |
| :--- | :--- |
| A1 | `104.[a].192.125` (*Locked*) |
| B1 | `104.[a].192.42` |
| C1 | `104.[a].192.260` |

A `/25` mask splits an IP range into two blocks of 128 addresses:
* **Block 0 (Lower Half):** `.0` to `.127` (Usable host addresses: `.1` to `.126`)
* **Block 1 (Upper Half):** `.128` to `.255` (Usable host addresses: `.129` to `.254`)

This means that `Interface C1` (`260`) is illegal: change it to a usable address between `.1` and `.126`.

### Example Solution Configuration

| Device | Interface | IP Address | Subnet Mask |
| :--- | :--- | :--- | :--- |
| **Host A** | `A1` | `104.198.[a].125` *(Locked)* | `255.255.255.128` *(Modified)* |
| **Host B** | `B1` | `104.198.[a].42` *(Modified)* | `255.255.255.128` *(Modified)* |
| **Host C** | `C1` | `104.198.[a].124` *(Modified)* | `255.255.255.128` *(Locked)* |

>[!NOTE]
>Replace `[a]` with the specific randomised number displayed on your screen (e.g., if `Host A` shows `104.198.42.125`, your network variable `[a]` is `42`).

## Theoretical Concepts

### Switched Network Topologies vs. Routers

A Network Switch operates at Layer 2 (Data Link layer) of the OSI model. It uses MAC addresses to forward frames within the same local network segment. Unlike a Layer 3 Router, a switch cannot route traffic across separate, distinct subnets. If devices connected to the same switch have mismatched subnet definitions or network IDs, they will drop each other's broadcast ARP requests and fail to communicate.

### Slicing with a /25 Subnet Mask

A mask of `255.255.255.128` means that 25 bits are assigned to the network ID, leaving 7 bits for hosts ($32 - 25 = 7$). 
* Total addresses per block: $2^7 = 128$.
* Because address spaces are broken exactly in half (`0-127` and `128-255`), a host at `.125` and a host at `.129` cannot see each other without a router, despite being plugged into the same physical switch. Keeping all host addresses strictly below `.127` avoids this boundary separation.
