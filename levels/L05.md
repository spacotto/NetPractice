## How to Solve Level 5

### Analyse the Router Interfaces

In this level, the router introduces static routing tables for the hosts. Begin by examining the two locked interfaces on `The Mighty Router` (`R`) to determine the network blocks required for Machine A and Machine B.
* Observe `Interface R1`, which connects to `Machine A`. It has a locked IP of `[a].[c].[e].126` and a locked mask of `255.255.255.128` (`/25`).
* Observe `Interface R2`, which connects to `Machine B`. It has a locked IP of `[b].[d].[f].254` and a locked mask of `255.255.192.0` (`/18`).

### Configure the Hosts

Assign IPs and subnet masks to the endpoint machines so they exist within the strict boundaries dictated by their respective router interfaces.
* Update `Interface A1`'s mask to exactly match `R1` (`255.255.255.128`). Set its IP address to a valid host in the same block. Since `R1` occupies `.126` (and `.127` is the broadcast address), set `A1` to `.125`, resulting in `[a].[c].[e].125`.
* Update `Interface B1`'s mask to exactly match `R2` (`255.255.192.0`). Set its IP address to a valid host within the `/18` block. Replicating the first three octets guarantees you remain in the correct block. Set the final octet to `.253`, resulting in `[b].[d].[f].253`.

### Configure the Routing Tables

For `Machine A` to successfully communicate with `Machine B` across the router, both endpoints must know exactly where to send traffic destined for outside networks. You must configure their routing tables to point to their local router interfaces as gateways.
* Examine `Machine B`'s routing table. The destination route is locked to `default` (which means any unknown network). Change the Gateway field from `192.168.0.254` to `Interface R2`'s exact IP: `[b].[d].[f].254`.
* Examine `Machine A`'s routing table. The destination route contains an invalid entry (`10..0.0.0/8`). Change this route to `0.0.0.0/0` (the numerical equivalent of `default`) to ensure it forwards all outside traffic to the router. Then, change its Gateway field to `Interface R1`'s exact IP: `[a].[c].[e].126`.

## Theoretical Concepts

### Default Gateways and the Routing Table

When a computer wants to send a packet to an IP address, it first checks its own subnet mask to see if the destination is on the local network (like a switch). If the destination is outside the local network, the computer must hand the packet off to a router. 

The routing table is the set of instructions telling the computer *which* router to use. The **Gateway** (or Next Hop) is simply the IP address of the router interface directly connected to that computer's local network.

### The Default Route (0.0.0.0/0)

Instead of maintaining a massive list of every single subnet in existence, endpoint machines (like laptops and servers) rely on a "catch-all" route. Writing `0.0.0.0/0` or `default` in the destination field tells the machine: *"If you do not know exactly where this IP is located locally, send it to this Gateway."* The router then takes responsibility for forwarding the packet to the next correct destination.
