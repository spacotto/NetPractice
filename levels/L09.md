# Level 9

!level9

## How to Solve Level 9

### Internet & External Subnet (A, B, R11)

The Internet dictates the IP block for the left side of the network (Hosts A and B).
* **The Internet Constraint:** Look at `Ir1` on the Internet router. It expects return traffic for the left network to go to `[e].198.[f].0/22`. This `/22` block (mask `255.255.252.0`) means the third octet can vary within a block of 4 (e.g., if `[f]` is 0, the valid subnets are `.0`, `.1`, `.2`, `.3`).
* **Router R1:** `R11` connects to the switch for A and B. Its mask is locked to `/25` (`255.255.255.128`). To satisfy the Internet's `/22` summary route, set `R11`'s IP to `[e].198.[f].1`.
* **Hosts A & B:** Both are connected to the same switch as `R11`.
    * Set `A1` IP to `[e].198.[f].2` and Mask to `255.255.255.128` (to match R11).
    * Set `B1` IP to `[e].198.[f].3` and Mask to `255.255.255.128`.
* **Routing (Left Side):**
    * Host A (`Ar1`): Set Destination to `0.0.0.0/0` and Gateway to `[e].198.[f].1` (R11).
    * Host B (`Br1`): Set Destination to `0.0.0.0/0` and Gateway to `[e].198.[f].1` (R11).

### Internet & Internal Subnet (C, D)

The Internet also dictates how it routes traffic back to the right side of the network (Hosts C and D).
* **The Internet Constraint:** Look at `Ir2`. It sends traffic for `10.0.0.0/27` to R1. This means the networks for C and D must be carved out of the `10.0.0.0` to `10.0.0.31` space.
* **Host C & Router R22:**
    * `C1`'s routing table (`Cr1`) is locked with Gateway `10.0.0.254`. This means `R22` (the router interface for C) MUST be `10.0.0.254`.
    * Wait, `10.0.0.254` is OUTSIDE the Internet's `10.0.0.0/27` constraint! The Internet route `Ir2` is a trick. Look at `Ir3`: a `default` route pointing to R1. As long as R1 knows how to reach C, the return traffic will use the default route.
    * Set `R22` IP to `10.0.0.254`. Since we need to conserve space for D, use a small mask. However, `R22`'s IP is `.254`, so it's at the end of a `/24`. Let's use `/24` (`255.255.255.0`).
    * Set `C1` IP to `10.0.0.1` and Mask to `255.255.255.0`.
* **Host D & Router R23:**
    * `R23` mask is locked to `/18`. `D1`'s destination route is `10.0.0.0/8`, but its gateway is locked to `[a].[b].[c].[d]`. This `[a].[b].[c].[d]` must be R23's IP.
    * Let's look at the given starting IPs for clues. `D1` starts with `[j].[k].19.131` and a `/26` mask. `R23` starts with `8.8.8.8`. Let's put them on a new subnet using the `[j].[k]` prefix.
    * Set `R23` IP to `[j].[k].19.129` and Mask to `255.255.255.192` (the `/26` from D1's default). *Correction: R23 mask is locked to /18 (255.255.192.0).*
    * If R23 mask is `/18`, we must use it. Set `R23` IP to `[j].[k].0.1`. Set `D1` IP to `[j].[k].0.2` and Mask to `255.255.192.0`.
    * Host D Routing (`Dr1`): The gateway is locked to `[a].[b].[c].[d]`. This means the interface for D1 MUST be on whatever network `[a].[b].[c].[d]` is. Let's look at the raw json: `{'hid':'D', 'rid':'Dr1', 'route':'10.0.0.0/8', 'gate':'[11-126]a.[1-254]b.[1-254]c.[1-254]d', 'route_edit':'true', 'gate_edit':'false'}`.
    * The gateway for D is LOCKED. You MUST set R23's IP to match the exact `[a].[b].[c].[d]` value shown on D's locked gateway field. Apply the locked `/18` mask to D1 as well.

### The Transit Link (R13 to R21)

Connect the two routers.
* `R21` mask is locked to `/30` (`255.255.255.252`). It starts with `[j].[k].18.253`.
* Let's use the `[j].[k].18.252/30` network.
* Set `R21` IP to `[j].[k].18.253`.
* Set `R13` IP to `[j].[k].18.254` and Mask to `255.255.255.252`.

### Router Routing Tables

Connect the subnets.
* **Router 2 (`R2r1`):** Needs a default route to R1.
    * Set Destination to `0.0.0.0/0`.
    * Set Gateway to `[j].[k].18.254` (R13).
* **Router 1:** Needs to know how to reach C and D via R2.
    * Route to C: Set `R1r1` Destination to `10.0.0.0/24` and Gateway to `[j].[k].18.253` (R21).
    * Route to D: Set `R1r2` Destination to the network of D. Look at the `[a].[b].[c].[d]` IP you assigned to R23. Calculate the network address using the `/18` mask. For example, if R23 is `11.22.33.44`, the `/18` network is `11.22.0.0/18`. Set Gateway to `[j].[k].18.253` (R21).
