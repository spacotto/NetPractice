## How to solve level 1

### Goal 1: *Host* **my PC** needs to communicate with host my little brother's computer.

To achieve Goal 1, we need to connect `host A` (my PC) to `host B` (my little brother's computer). Because `Host B`'s interface (`B1`) is locked, we must modify `Host A`'s interface (`A1`) to match `Host B`'s network parameters. The **subnet mask** `255.255.255.0` dictates that the first three octets (`104.98.23`) represent the **network ID** and must be identical on both machines. **Only the last digit can be unique to identify the specific host**. Thus, to join `Host B`'s network, `Host A`'s IP must **start** with `104.98.23.`, and **end** with any **valid host number from 1 to 254** (**excluding 12**, which `Host B` is already using).

- [x] **STEP 1:** Change the second octet from `93` to `98` to match the network ID.
- [x] **STEP 2:** Change the last octet (`339`) to  any **valid host number from 1 to 254** (**excluding 12**, which `Host B` is already using).

### Goal 2: *Host* **my Mac** needs to communicate with host my little sister's computer.

To achieve Goal 2, we need to connect `host C` (my Mac) to `host D` (my little sister's computer). Because `Host C`'s interface (`C1`) is locked, we must modify `Host D`'s interface (`D1`) to match `Host C`'s network parameters. The **subnet mask** `255.255.0.0` dictates that only the first two octets (`211.191`) represent the **network ID** and must be identical on both machines. The remaining two numbers can vary between the hosts. Thus, to join `Host C`'s network, `Host D`'s IP must start with `211.191.`. The last two octets can be **any valid numbers between 0 and 255 (as long as the entire combination doesn't exactly duplicate Host C's IP)**.

- [x] **STEP 1:** Change the second octet from `190` to `191` to match the network ID.
- [x] **STEP 2:** Change the third octet (`282`) to  any **valid host number from 1 to 254** (**excluding 161**, which `Host C` is already using).
